03 / Core Concept

Paging — In Depth

Fixed-size memory blocks for efficient allocation

Paging divides a process into pages and physical memory into equal-sized frames. Pages are mapped to frames anywhere in RAM using a Page Table. Page Size = Frame Size.
📊 Pages → Mapping → Main Memory

Pages are mapped to frames anywhere in Main Memory — no external fragmentation!

✅
Advantages
✅ Efficient memory usage
✅ Supports Virtual Memory
✅ No External Fragmentation
✅ Easy process swapping
❌
Disadvantages
❌ Internal Fragmentation possible
❌ Overhead of maintaining Page Tables

Address Translation in Paging

How a logical address is translated to a physical address

🔢
Logical Address

Generated by the CPU. Split into two parts: Page Number (p) and Page Offset (d).

📋
Page Table Lookup

Page number p indexes into the Page Table to fetch the Frame Number (f).

📍
Physical Address

Physical address = Frame Number × Frame Size + Offset (d). This points to the exact memory location.

Memory Management Unit (MMU)

The mapping from virtual to physical address is done by the MMU — a hardware device. This mapping process is known as the paging technique.

📦 Physical memory → fixed-size Frames
📄 Logical memory → fixed-size Pages
⚖️ Page Size = Frame Size
🧮 Address Bit Formula
Logical Space = 128M = 227 words
→ Logical Address = 27 bits
Physical Space = 16M = 224 words
→ Physical Address = 24 bits
07 / Page Table & Virtual Memory

Page Table Structure & Virtual Memory

Types of page tables and the concept of virtual memory

🗂 Structure of the Page Table

Modern systems have massive address spaces (232 to 264), making a single page table too large for contiguous memory.

A. Hierarchical Paging

Page table broken into multiple levels. Only the outer page table needs to stay in memory at all times.

B. Hashed Page Tables

Virtual page number hashed into a table. Each entry holds a linked list to handle collisions. Used for >32-bit spaces.

C. Inverted Page Table

One table for the entire system — one entry per real frame.
✅ Less memory  |  ❌ Slower search

💾
Virtual Memory in OS

A technique that gives programs the appearance of large, continuous memory even when RAM is limited. Pages are moved to/from disk as needed.

🎯 Objectives of Virtual Memory
✅Program doesn't need to be fully loaded — only needed parts run.
✅Programs can be larger than physical RAM available.
✅Creates the illusion of large memory even if actual RAM is small.
✅Uses both RAM and disk — loads only parts of programs as needed.
✅Allows system to run more programs at once more efficiently.
